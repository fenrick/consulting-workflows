# Brief Storyline Tracking Templates

Use these templates to keep the storyline reviewable.

- `storyline-architecture-template.md`: the core artifact; use for the governing question, top-level answer, pillars, support logic, and proof gaps
- `open-questions-template.md`: use for missing evidence, weak logic, and unresolved objections
- `decision-log-template.md`: use for major framing choices, dropped alternatives, and risk tradeoffs

Keep the files in `tracking/` in the target repo. Update them during the work, not only at the end.
