# Input Evidence Catalog Tracking Templates

## Purpose

These templates keep intake evidence structured enough for audit, storyline, and drafting work.

## Template roles

- `source-register-template.md`: use for file identity, provenance, status, role, and supersession
- `review-notebook-template.md`: use for factual extraction, confidence, and cross-source insights
- `open-questions-template.md`: use for missing metadata, unreadable files, and contradictions
- `document-map-template.md`: use for folder/archive structure and replacement mapping

## Working rule

Keep live files under `tracking/` and `inputs/processed/`. Update them during intake.
