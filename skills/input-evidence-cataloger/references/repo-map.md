# Input Evidence Catalog Repo Map

## Main files

- `tracking/source-register.md`: one row per source with provenance and status
- `tracking/review-notebook.md`: factual extractions and cross-source observations
- `tracking/open-questions.md`: unresolved metadata or evidence issues
- `inputs/processed/document-map.md`: input structure and supersession map

## Templates

- `assets/templates/source-register-template.md`
- `assets/templates/review-notebook-template.md`
- `assets/templates/open-questions-template.md`
- `assets/templates/document-map-template.md`

## Typical flow

1. Inventory `inputs/`.
2. Build the document map.
3. Extract facts into the review notebook.
4. Record open questions.
5. Hand off to audit or storyline work.
