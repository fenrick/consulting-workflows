# Consulting Workflow Tracking Templates

## Purpose

These templates keep the workflow state explicit enough that another operator can pick up the engagement without reconstructing it from chat.

## Template roles

- `workflow-runbook-template.md`: use for stage objectives, inputs, outputs, blockers, owners, and gate results
- `workflow-status-template.md`: use for compact status reporting across the active stage path

## Working rule

These templates belong in `tracking/` in the target repo. Keep both files current throughout the run.
