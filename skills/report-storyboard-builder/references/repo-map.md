# Report Storyboard Repo Map

## Purpose

This map shows where report structure planning should live before detailed prose drafting starts.

## Main files

- `tracking/report-storyboard.md`: live section-level storyboard
- `tracking/open-questions.md`: unsupported sections and proof gaps
- `tracking/storyline-architecture.md`: upstream narrative baseline
- `report-body/`: downstream draft section files

## Templates

- `assets/templates/report-storyboard-template.md`
- `assets/templates/section-file-plan-template.md`
- `assets/templates/open-questions-template.md`

## Typical flow

1. Start from the validated storyline.
2. Build the section storyboard.
3. Produce the section file plan.
4. Hand off to document drafting.
