# Report Storyboard Quality Standard

## Purpose

This file defines the quality bar for turning a storyline into a report structure that writers can execute reliably.

Use it during section planning, resequencing, and handoff.

## Core standard

- Every section has a purpose, claim, and proof obligation.
- Sections do not materially overlap.
- Unsupported sections are explicit.
- Drafting order supports the narrative.

## Authoring standard

- Build from skeleton to bullets to short notes.
- Keep section labels plain and stable.
- Avoid polished prose and recommendation drift at this stage.
- Humanise awkward phrasing without changing logic.

## Tracking standard

- `tracking/report-storyboard.md` is the live baseline.
- Unsupported sections stay in `tracking/open-questions.md`.
- The section file plan must remain aligned with the current storyboard.
- Front-end requirements and appendix candidates must be visible in the live planning files.

## Handoff standard

- Drafting should be able to start without guessing section purpose or evidence burden.
- Any section that cannot yet be supported must already be marked as a gap.
- Section order must hold together when read as a full report path, not only as isolated sections.
