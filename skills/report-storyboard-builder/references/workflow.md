# Report Storyboard Workflow

## Purpose

This workflow turns a validated storyline into a report structure that writers can execute without guessing.

The output is section logic, proof obligations, exclusions, and drafting order.

## 1. Set up the working files

Create or refresh:

- `tracking/report-storyboard.md`
- `tracking/open-questions.md`

## 2. Lock the report objective

Record:

- the report audience
- the report purpose
- required sections or appendices
- what the report is not trying to do
- any packaging or delivery constraints

## 3. Build the section skeleton

Start with:

1. front end requirements
2. core body sections
3. appendix candidates

Then define one clear purpose per section.

## 4. Expand to section bullets

For each section, capture:

- section title
- purpose
- key claim
- proof obligation
- required evidence
- practical implication
- exclusions
- dependencies

## 5. Write the storyboard notes

Once the section bullets hold together, add short prose notes where needed to explain:

- the section order
- where the argument turns
- why sections are separated
- where evidence is currently thin

## 6. Run the authoring loop

Use this cycle:

1. section skeleton
2. section bullets
3. short notes
4. narrative-flow pass
5. humanise awkward machine phrasing
6. restore neutral voice and simple language

## 7. Finalise the handoff

Before handoff:

- `tracking/report-storyboard.md` is current
- `tracking/open-questions.md` captures unsupported sections
- the section file plan is ready for `report-body/`
- `references/validation-checklist.md` has been run
