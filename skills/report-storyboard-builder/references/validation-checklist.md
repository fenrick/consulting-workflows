# Report Storyboard Validation Checklist

## Purpose

Use this checklist to confirm that the report structure is usable by writers and defensible against the storyline and evidence base.

## How to use this checklist

Run it after the first complete section map and again after major resequencing.

- Check section logic before sentence quality.
- Check evidence obligations before handoff.
- Check writing quality last.

## Section logic checks

- Every section has one clear purpose.
- Every section has a key claim or an explicit descriptive role.
- Proof obligations are stated.
- Overlap between sections was removed.
- Drafting order makes sense end to end.

## Evidence checks

- Each section has required evidence or an explicit evidence gap.
- Unsupported sections are logged in `tracking/open-questions.md`.
- Front-end requirements and appendix candidates are explicit.

## Writing checks

- The artifact was built through skeleton, bullets, short notes, and flow passes.
- Language is plain and neutral.
- Humanisation removed machine habits without changing structure.
