# Workshop Write-up Workflow

## Purpose

This workflow turns workshop transcripts and artifacts into a formal write-up that preserves what actually happened.

It is a reconstruction workflow, not a fiction generator.

## 1. Set up the working files

Create or refresh:

- `tracking/workshop-writeup.md`
- `tracking/workshop-evidence-map.md`
- `tracking/open-questions.md`
- `tracking/decision-log.md`

## 2. Lock the workshop frame

Record:

- the workshop purpose
- the session date and structure
- known participants and roles
- artifact sources in scope
- any missing transcript or artifact gaps

## 3. Build the write-up skeleton

Start with:

1. purpose and scope
2. participants and session structure
3. key themes
4. decisions made
5. action items
6. unresolved issues

## 4. Expand to evidence bullets

For each theme, decision, or action, capture:

- what was said or produced
- the transcript or artifact source
- confidence level
- owner or participant if known
- unresolved ambiguity

## 5. Write the workshop narrative

Once the bullets are stable, write short paragraphs that:

- reconstruct the discussion plainly
- preserve disagreement where it mattered
- separate decisions from tentative ideas
- keep outputs and actions tied to evidence

## 6. Run the authoring loop

Use this cycle:

1. write-up skeleton
2. evidence bullets
3. short paragraphs
4. narrative-flow pass
5. humanise awkward machine phrasing
6. restore neutral voice and simple language

## 7. Finalise the handoff

Before handoff:

- `tracking/workshop-writeup.md` is current
- `tracking/workshop-evidence-map.md` traces major claims to evidence
- unresolved issues stay in `tracking/open-questions.md`
- confirmed decisions stay in `tracking/decision-log.md`
- `references/validation-checklist.md` has been run
