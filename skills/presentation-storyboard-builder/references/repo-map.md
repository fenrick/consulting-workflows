# Presentation Storyboard Repo Map

## Purpose

This map shows where storyboard planning should live so the narrative sequence does not get buried in deck-production files.

## Main files

- `tracking/presentation-storyboard.md`: the live slide-by-slide storyboard
- `tracking/open-questions.md`: unsupported slide claims and follow-ups
- `tracking/storyline-architecture.md`: upstream narrative baseline
- `tracking/source-register.md`: evidence source map

## Templates

- `assets/templates/presentation-storyboard-template.md`
- `assets/templates/slide-map-template.md`
- `assets/templates/open-questions-template.md`

## Typical flow

1. Start from the validated storyline.
2. Build the slide map and storyboard.
3. Remove weak or duplicate slides.
4. Handoff for deck production.
