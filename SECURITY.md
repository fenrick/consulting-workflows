# Security Policy

## Reporting a vulnerability

Open a private security advisory if your host platform supports it.
If private advisories are not available, do not file a public issue with exploit details.

Include:

- affected files/skill names
- impact and expected abuse path
- minimal reproduction
- suggested mitigation (if available)

## Support window

- Latest tagged release: supported
- Previous minor release: best effort
- Older releases: no guaranteed fixes

## Secrets and data handling

- Do not commit credentials, tokens, or client data.
- Treat all sample input documents as non-sensitive unless explicitly scrubbed.
